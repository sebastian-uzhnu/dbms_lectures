# MySQL: Робота з JSON

MySQL — це реляційна база даних (таблиці, рядки, стовпці). Але починаючи з версії 5.7, вона чудово вміє працювати з **JSON**.
Це означає, що ми можемо мати найкраще з двох світів: надійність SQL та гнучкість NoSQL.

---

## Навіщо пхати JSON у MySQL?

Уявіть, що ви робите інтернет-магазин техніки.

- У **ноутбука** є: `процесор`, `RAM`, `SSD`.
- У **футболки** є: `розмір`, `колір`, `тканина`.
- У **книги** є: `автор`, `ISBN`, `обкладинка`.

Як це зберегти в одній таблиці `products`?

1.  **Погана ідея:** Створити 100 стовпців (`attr_size`, `attr_color`, `attr_cpu`...), більшість з яких будуть пусті.
2.  **Класична ідея (EAV):** Створити окрему таблицю `attributes` і зв'язувати її. (Складно і повільно).
3.  **Сучасна ідея:** Зробити один стовпчик `attributes` типу `JSON` і класти туди все, що завгодно!

## Створюємо таблицю з JSON

```sql
CREATE TABLE products (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(255),
    price DECIMAL(10, 2),
    details JSON -- Ось наша магічна колонка
);
```

Тепер додамо дані. Дивіться, як відрізняється структура в `details`!

```sql
INSERT INTO products (name, price, details) VALUES
('MacBook Pro', 2500.00, '{"cpu": "M1 Max", "ram": "32GB", "ssd": "1TB"}'),
('Nike T-Shirt', 30.00, '{"size": "L", "color": "black", "material": "cotton"}'),
('Harry Potter', 15.00, '{"author": "J.K. Rowling", "pages": 500}');
```

## Як шукати по JSON?

Тут починається магія. Ми використовуємо спеціальний синтаксис: стрілочку `->` або `->>`.

### 1. Витягнути значення

Оператор `->>` витягує значення і **прибирає лапки** (робить його звичайним текстом або числом).

```sql
-- Показати назву товару і його колір (якщо є)
SELECT name, details->>'$.color' as color
FROM products;
```

**Результат:**
| name | color |
|---|---|
| MacBook Pro | NULL |
| Nike T-Shirt | black |
| Harry Potter | NULL |

### 2. Фільтрувати по значенню

Знайдемо всі футболки розміру "L".

```sql
SELECT * FROM products
WHERE details->>'$.size' = 'L';
```

### 3. Глибокий пошук (Вкладені об'єкти)

Якщо у вас складний JSON, наприклад: `{"specs": {"screen": "15"}}`.

```sql
SELECT * FROM products
WHERE details->>'$.specs.screen' = '15';
```

## Як змінювати JSON?

Ми можемо змінювати окремі поля всередині JSON, не переписуючи весь рядок!

### Додати або оновити поле `JSON_SET`

Змінимо RAM у Макбука на 64GB.

```sql
UPDATE products
SET details = JSON_SET(details, '$.ram', '64GB')
WHERE name = 'MacBook Pro';
```

### Видалити поле `JSON_REMOVE`

Приберемо інформацію про матеріал футболки.

```sql
UPDATE products
SET details = JSON_REMOVE(details, '$.material')
WHERE name = 'Nike T-Shirt';
```

## Продуктивність (Індекси)

"А це не буде працювати повільно?"
Якщо ви просто шукаєте по JSON (`WHERE details->>'$.id' = 123`) — так, це повільніше, ніж звичайна колонка, бо база сканує всі рядки.

Але у MySQL є лайфхак: **Virtual Columns (Віртуальні стовпці)**.
Ми можемо створити "фейковий" стовпчик, який автоматично витягує дані з JSON, і повісити на нього індекс!

```sql
-- 1. Створюємо віртуальну колонку для кольору
ALTER TABLE products
ADD COLUMN color VARCHAR(50)
GENERATED ALWAYS AS (details->>'$.color') STORED;

-- 2. Створюємо індекс на цю колонку
CREATE INDEX idx_color ON products(color);
```

Тепер пошук по кольору буде таким же миттєвим, як і в звичайній таблиці!

## Коли використовувати JSON?

- **ТАК:** Коли атрибути товару сильно відрізняються (одяг, техніка, їжа).
- **ТАК:** Для налаштувань користувача, які часто змінюються.
- **ТАК:** Для логів та даних від API.
- **НІ:** Для зв'язків (не пишіть `user_ids: [1, 2, 3]`). Використовуйте таблицю зв'язків!
- **НІ:** Для даних, по яким ви постійно робите аналітику і JOIN-и.

Використовуйте JSON з розумом, і ваша база буде літати!
